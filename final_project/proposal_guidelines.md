# Project Proposal Guidelines
The final project for this course will be a significant coding project that applies the numerical methods we are learning to a problem of your choice. The project will be due on the day of the final exam, but you will be required to submit a project proposal by March 23, 2026.

## Selecting a Topic
Before starting the proposal, you should select a topic that interests you and is relevant to the course material. This could be a problem from your major, a research question you have, or a real-world application of numerical methods. You should also consider the scope of the project and ensure that it is feasible to complete within the time frame.

You can find links to examples of projects from similar classes in project_inspo.md. You can also find inspiration from your other classes, research papers, or real-world problems.

## Set up a GitHub Repository
Once you have selected a topic, you should [set up a GitHub repository](https://docs.github.com/en/repositories/creating-and-managing-repositories/quickstart-for-repositories) for your project. You will submit a link to this repository to satisfy the project proposal requirement, and you will use this repository to develop your project over the course of the semester. 

## Writing the Proposal
In your new repository, you should create a README.md file that includes the following:
- project title
- a brief description of the project and its goals, including the numerical methods you plan to use (~1-2 paragraphs)
- a list of the planned directory structure and files you plan to include in the repository (e.g. code, data, documentation, results). Here, you should think about the different components of your project and how you will organize them.
- a list of any resources you plan to use (e.g. papers, datasets, libraries)
- Several bullet points outlining the steps you plan to take to complete the project. This should include a rough timeline for when you plan to complete each step. You should also include one or two Reach Goals that you would like to achieve if you have extra time after completing the main goals of the project.
- any other notes or references necessary to understand the proposed project

If you are working as a group, you should include the names of all group members in the proposal and outline how you plan to divide the work among group members. (No more than three people per group.)

As you progress through the project, you should update your README.md file to reflect any changes in your plans or goals. This will help you stay organized and on track, and it will also provide a clear record of your project development process.

## Submitting the Proposal

Once you are done, ensure that your proposal is committed and pushed to your GitHub repository. Then, submit the link to your repository in [this Moodle assignment](https://moodle.adelphi.edu/mod/assign/view.php?id=5517828).
